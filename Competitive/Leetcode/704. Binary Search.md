Given an array of integers **nums** which is sorted in ascending order, and an integer **target**, write a function to search **target** in **nums**. If **target** exists, then return its index. Otherwise, return -1/

You must write an algorithm with $O(log n)$ runtime complexity

# Approach
<!-- Describe your approach to solving the problem. -->
1. Set Upper and Lower bound, each defined by Maximum Index and Zero (0), respectively

2. First, check if the middle element is already the target 
3. If not, compare and focus on the appropriate half of the vector
4. Repeat until target found and return its index.

# Complexity
- Time complexity:
$O(log n)$, could be $O(1)$ since it also checks firsthand if the middle is already the target

- Space complexity:
$O(1)$ at best, $O(log n$ as usual.

# Code
```cpp []
class Solution {
public:
    int search(const vector<int>& nums, int target) {
        int min = 0, max = nums.size() - 1;

        while (min <= max) {
            int mid = min + ((max - min) >> 1);

            if (nums[mid] == target) return mid;

            if (target > nums[mid]) {
                min = mid + 1;
            } else {
                max = mid - 1;
            }
            
        }
        return -1;
    }
};
```
